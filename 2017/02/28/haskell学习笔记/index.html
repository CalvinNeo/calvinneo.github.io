Ch3 Types and Classes

  1. 表达式的类型
    如果一个表达式e的计算得到类型是t的结果，那么称为e具有类型(has type)t，写作

    e :: t

    由于Haskell是静态类型的(static typed)，所以编译器能够通过类型推导(type inference)得到所有表达式的类型

  2. :t或:type
    使用:t或:type可以获得参数的类型

  3. Haskell的基本类型

    Bool          | 布尔                                                   
Char          | 字符，字符直接量用单引号括起                           
String        | 字符串，字符串直接量用单引号括起，也可以看做Char的List 
Int           | 固定精度整数
Integer       | 任意精度整数
Float         | 浮点数
[t]           | 类型t的列表
(t1, ..., tn) | 元组

    特别地，使用中括号括起数据生成列表时，中括号称为data constructor，括起类型生成列表类型是，称为type constructor

  4. 函数的类型
    函数也就是映射，Haskell中使用t1 -> t2表示函数将t1类型映射到t2类型
    这里是使用的type form表示的函数，也可以使用value form来表示函数，做法是在最前面加一个反斜杠，\x -> y，这其实就是Haskell里面的lambda表达式，对于多元的lambda表达式，可以写成\x y -> x + y
    由此看出，由于Haskell的类型推导机制，函数的type form在具备value form是并不是强制的，这和C++等命令式语言强制性的函数签名是不一样的。当然有时候会写上type form用来限定参数的类型

  5. unit
    ()称为unit，类似于void，会在monad中用到

  6. 柯里化
    对于一个多元函数，例如div，设想它的类型是这样的

    add :: (Int, Int) -> Int
add = \(x, y) -> x + y

    但实际上它的类型是这样的

    :t div
div :: Integral a => a -> a -> a

    事实上这是因为add x y实际上接受一个参数a，并返回一个一元函数（闭包），这个函数的类型是a -> a。这个一元函数的作用是将自己的参数b和add的参数a相加，这样的函数称为柯里函数
    因此实际上div的类型是

    div :: Integral a => a -> (a -> a)

    注意到Haskell中的->符号时右结合的，因为柯里化是从左边开始的，所以可以省略括号写成a -> a -> a
    使用lambda可以写为

    add = \x -> (\y -> x + y)
add x = \y -> x + y

  7. 多态函数
    Haskell中的函数默认可以接受任意类型的参数，但有时需要限定类型，例如sum函数，去接受一个Bool类型是没有意义的，所以使用sum :: (Num a) => [a] -> a来为a提供类型约束，称之为type classes，类似于C#中接口(interface)，这在Ch2.5 自定义函数一节中已经提到过
    常用的type classes有

    • Num：表示数字
    • Eq：表示能判断相等的类型
    • Ord：表示能比较大小的类型
      特别注意，Ordering是一个类型，包含GT、LT、EQ三个constructor
    • Show：可以被转成字符串的类型，Haskell中除了函数都能被表示成字符串
    • Read：可以从字符串被读取的类型
    • Enum：能被枚举的类型
    • Bounded：有界的类型
    • Integeral：整型
    • Floating：浮点型

  1. if
    Haskell中的if语句和其他语言的并无二致

    但是else分支是必须的，毕竟是强类型语言

    这个名字还挺熟悉的，原来是在程序设计语言原理上看到过，当时翻译成守卫
    这里的guard指的是|符号，注意|在list comprehension另有用途

    abs n | n >= 0 = n
      | otherwise = -n

    pattern matching是类似这样的东西

    not :: Bool -> Bool
not True = False
not False = True

    这可以借助OOP中的虚函数(dynamic dispatch)实现，例如下面的C#伪代码

    特别地，下划线_表示可以匹配任何参数，称为wildcard。例如

False && _ = False

  4. lazy evaluation
    lazy evaluation相对于eager evaluation有相当的好处，阐明这点将贯穿整个课程，这里讲了一个比较具体的例子

    f x = 4711

    于是可以有下面的求值方案，这里X是某个任意值

    f(True && X)
= f(X)
= 4711

    这样的好处是避免了对X的求值（注意这里X没有被逻辑与短路），因为X可能是不可计算的：假如定义并求值X = X或者X = head []，那程序是不能终止(non-terminating)的，在Haskell中是通常的一种错误(Error)形式。但是f(X)是可以计算的。采用eager evaluation的大多数其他语言会在True && X表达式时就对X求值，而这时会造成问题的。
    此外，lazy evaluation无论对于什么样的求值顺序结果都是不变的，这是由于Haskell语言本身（除了Monad）不会造成副作用。例如对于上面的例子可以直接根据f x = 4711归约

    f(True && X) = 4711

  5. pattern具有优先级
    对于这个例子中，值永远是False，因为第一个规则的优先级更高

    _ && _ = False
True && True = True

  6. 重复的名字不能出现在pattern中
    此外还需要注意重复的名字(patterns may not repeat variables)，例如下面的代码是错误的

    b && b = True

    会输出错误

    ? Conflicting definitions for ‘b’
Bound at: F:\Codes\Haskell\3.hs:2:1
        F:\Codes\Haskell\3.hs:2:6
? In an equation for ‘Main.&&’

    这是为什么呢？
    其实这样的东西称为nonlinear pattern。理想情况下，我们希望通过使用同一个字符b来限定&&的左操作数和右操作数是相等的。但是这是不可能实现的，因为当左右操作数出现lambda时，是没有一个统一标准判断lambda是否相等的

  7. list pattern和:
    运算符:可以将元素添加到列表前面，下面两行的效果是等价的

    [1,2,3]
1:(2:(3:[]))

    然而它还可以用在pattern matching中，例如实现head和tail

    head :: [a] -> a
head (x:_) = x
tail :: [a] -> [a]
tail (_:xs) = xs

    注意点：

    1. 用x:xs去pattern matching空list是未定义行为
    2. x:xs一定要用括号括起来，因为函数调用(application)的优先级比:的优先级要高

    3. 注意区分**(x:y)**和**[x:y]**
      [x:y]匹配具有一个参数的list，这个参数满足匹配x:y
      (x:y)匹配具有x作为head和y作为tail的list
      总之，[]匹配是固定个数的，可以利用这个来匹配终结条件，例如

      product [] = 1
product (x:xs) = x * product xs

      或

      product [x] = x
product (x:xs) = x * product xs

  8. integer pattern
    这是一个类似数列递推公式的pattern，可以使用n + k这样的形式使用。这时候n是一个变量，而k是一个常数

    pred :: Int -> Int
pred (n + 1) = n

fib :: Integer -> Integer 
fib 0 = 1
fib 1 = 1
fib n = fib (n-1) + fib (n-2)
main = print (fib 5)

    函数调用(application)的优先级比+等代数运算的优先级要高，所以同样要用括号括起来

  9. lambda
    在上一Chapter中通过add的例子讲到Haskell如何通过柯里化让一个函数返回另一个函数，而使用lambda可以更清晰地表现出一个函数究竟是返回的一个值还是另一个函数。例如

    const :: a -> b -> a
const x _ = x

    也可以写成柯里形式，接受一个参数，返回一个接受一个参数的函数

    const x = \_ -> x

    使用lambda同时还可以声明一个匿名函数，然后避免定义这个只用一次的具名函数

Ch5 List Comprehensions

顾名思义，这章讲的是Haskell的列表生成器

[x + 1 | x <- [1..5]]

可以生成笛卡尔积

> [(x,y) | x <- [1,2,3], y <- [4,5]]
[(1,4),(1,5),(2,4),(2,5),(3,4),(3,5)]

这类似嵌套的for循环的性质，越往右for的嵌套层数越深

> [(x,y) | y <- [4,5], x <- [1,2,3]]
[(1,4),(2,4),(3,4),(1,5),(2,5),(3,5)]

同样类似嵌套的for循环，内层for的界可以依赖外层的值

> [(x,y) | x <- [1..3], y <- [x..3]]

它还可以做

concat :: [[a]] -> [a]
concat xss = [x | xs <- xss, x <- xs]

这类似于派通(Python)中的

[item for sublist in [[1,2], [3], [4,5,6]] for item in sublist]

还可以加上guard

[x | x <- [1..10], even x]

下面是一个质数暴力筛程序

factors :: Int -> [Int]
factors n = [x | x <- [1..n], n `mod` x == 0]
prime :: Int -> Bool
prime n = factors n == [1,n]
primes :: Int -> [Int]
primes n = [x | x <- [2..n], prime x]

Ch7 Higher-Order Functions

上一章讲了如何使用递归来实现某些函数，这章学习通过高阶函数如map、filter、foldr来实现它们
高阶函数(Higher-Order Functions)指的是接受一个函数作为参数，或者返回一个函数的函数（然而Haskell天生自带Currying，所以参数大于等于2就一定是高阶函数了啊）

  1. 介绍一些关于Combinator的书

  2. 用途

    1. common programming idiom
      写出更短小精炼的程序

    2. DSL
      可以方便地定义领域特定语言(DSL)，例如一个简单的加法器的语法

      data expr
    = value
    = add expr expr

      然后再实现一个eval函数即可

      eval :: expr -> Int

      要是C++的话可能会想着自己撸个逆波兰式或者LL解释器啥的
      另外Haskell的模式匹配也在这方面有比较大的作用

    3. algebra properties
      高阶函数具有一定的代数性质可以进行宏观变换，而不要担心实现细节，类似于上一章讲到的分为大操作小操作
  3. map
    map接受一个函数和一个列表，并返回一个列表
    map :: (a -> b) -> [a] -> [b]
    注意不要错写成
    map :: a -> b -> [a] -> [b]
  4. filter
    可以通过guard或者list comprehension来实现filter

  5. fold
    Haskell或者C++17中的fold expression，类似于python等语言中的reduce函数。这两者的区别在reduce只能处理一个相同类型（fold前），但是fold可以处理两个不同类型（fold前和fold的结果）
    Haskell中分为foldl和foldr
    foldr将归约到的值作为运算f的右操作数

    foldr :: Foldable t => (a -> b -> b) -> b -> t a -> b
foldr f v [] = v
foldr f v (x:xs) = f x (foldr f v xs)

    它接受三个参数：
    (a -> b -> b)：一个接受a和b类型参数并返回b类型参数的函数
    b：归约到的类型
    t：一个Foldable的类型
    t a：这个用法应该在后面会被讲到，可以近似理解成一个接受一个类型参数a的 t类型。例如，当t为list时，t a就是[a]
    因此这个函数可以理解为

    foldr f 列表中的每一项 当前归约的结果

    reduce可以用fold来实现，例如

    print reduce(lambda x, y: x + [y], [1,2,3,4], [])

    可以用foldr实现为

    main = print (foldr (\x -> \y -> [x] ++ y) [] [1,2,3,4])

    类似的有foldl函数

    foldl :: Foldable t => (b -> a -> b) -> b -> t a -> b

    上面的reduce可以用foldl实现为

    main = print (foldl (\x -> \y -> x ++ [y]) [] [1,2,3,4])

  6. 使用fold实现函数
    fold是被提炼出的一种“大操作”，利用foldr可以实现很多函数，之前这些函数往往要递归地通过guard、if或者list comprehension来实现

    sum = foldr (+) 0

    这是因为

      sum [1,2,3]
= foldr (+) 0 [1,2,3]
= foldr (+) 0 (1:(2:(3:[])))
= 1+(2+(3+0))

    类似地可以定义

    product = foldr (*) 1
or = foldr (||) 1
and = foldr (&&) 1

    对于length函数可以如此定义

    length = foldr (\_ n -> 1 + n) 0
length = sum . map (\_ -> 1)

    对于reverse函数可以如此定义

    reverse = foldr (\x xs -> xs ++ [x]) []

    这是因为

      reverse [1,2,3]
= reverse (1:(2:(3:[])))
= (([] ++ [3]) ++ [2]) ++ [1]

    所以可以看到foldr接受a类型的x和b类型的xs，然后反向地去连接xs和[x]，最后得到的是另一个列表，这和上面的length函数是不一样的
    特别地，可以重新定义++

    (++ ys) = foldr (:) ys

    这是因为

    (xs ++ ys) = foldr (:) ys xs

    这表示把Foldable类型的xs中的每个元素:到ys的前面，这就相当于

      foldr (:) ys [1,2,3]
= foldr (:) ys (1:(2:(3:[])))
= (1:(2:(3:[ys])))

    继续改写 

      (xs ++ ys)
= (++) xs ys -- 这里原版写的是(++) ys xs但是我觉得应该是(++) xs ys
= foldr (:) ys xs

    因此可以得到

      (++) ys = foldr (:) ys
= (++ ys) = foldr (:) ys

  7. fold的作用

    1. 有些递归函数用foldr更容易实现（可以从之前的讨论中看出）
    2. 有些函数的性质可以通过foldr的代数性质推导得到，例如fusion规则和banana split规则
      • fusion
        fusion(composition)是之前讨论过的.运算符，用来实现复合函数，例如
        length = sum . map (\_ -> 1)
        这同样是因为Haskell是lazy的，所以看到map时，并不立即计算map
        特别地，.也可以理解为一个函数，它的定义可以看做
        (.) :: (b -> c) -> (a -> b) -> (a -> c)
f . g = \x -> f (g x)
      • banana split

    3. 使用foldr而不是递归，可以方便进行优化

Ch8 Functional Parsers

在官网上下载的PPT中第8章是Declaring Types and Classes，但实际上这章在课程中被放到了第10章。本章是Functional Parsers，会讲授重要的概念Monads，于是教授穿了件自己设计的（密集恐惧症）衣服
这一章会有点突兀，有比较多的没学过的东西
首先要补充一个控制结构，对于下面的guard

f 0 = 18
f 1 = 15
f 2 = 12
f x = 12 - x

这种写法又称为piece-wise version，因为实际上定义了4个f函数。如果用case，可以写作

f x = 
case x of
    0 -> 18
    1 -> 15
    2 -> 12
    _ -> 12 - x

  1. 介绍Functional Portal

  2. Parser Type
    很自然地，在Haskell中，用函数来表示Parser

    type Parser = String -> Tree

    但是有时候Parser只处理输入串String的一部分，所以Parser也要返回未使用的String

    type Parser = String -> (Tree, String)

    有时候输入串String不能生成语法树Tree，于是Tree并不是一定有的，是一个optional值，这在Haskell中可以用Maybe来表示

    data Maybe a = Nothing
             | Just a

    这是Haskell用来处理optional值的方法，data语句在第10章会有介绍，这里可以近似地理解为C++中的enum，Nothing和Just a是Maybe a的两个constructor。
    Nothing很好理解，类似于其他语言中的null、None、nil
    Just是个比较奇怪的概念，他表示表示不是Nothing，如果一个类型可以从Nothing构造，那么返回一个非Nothing的值a时就需要返回Just a，这有点类似于函数重载或者SFINAE的意味。
    但是实际处于方便考虑，并不需要使用Maybe这东西，用一个简单的list就可以了

    type Parser = String -> [(Tree, String)]

    因此这个list要不是一个单元素的列表(singleton list)，要不是个空列表
    最后，Parser并不一定返回一个语法树，所以加上一个泛型

    type Parser a = String -> [(a, String)]

    总结一下，现在我们已经知道了list的作用是允许optional，tuple的作用是辅助类似于遍历字符串的一个指针。
    事实上这个Parser是一个Monad雏形。注意如果使用附带的代码Parsing.hs会发现它对Parser的定义不尽相同，这会导致运行视频中的代码可能会出现错误（接下来会看到）。视频中给出上面这样的Parser定义是为了方便理解的考虑，

  3. 一些简单的语法分析器
    使用附带的Parsing.hs运行下面的代码可能会报错，因为使用了newtype定义的类型和[(a, String)]是两个完全不一样的类型了，所以要在前面加一个P。可以参考Parsing.hs里面的item和parse函数

    1. item
      这个Parser在输入为空是fail，否则读取输入的第一个字符，下面根据type Parser a = String -> [(a, String)]来推导item的类型
      item :: Parser Char
     :: String -> [(Char, String)]
     :: [Char] -> [(Char, [Char])]
      下面查看item的定义
      item = \inp -> case inp of
                    [] -> []
                    (x:xs) -> [(x,xs)]
    2. failure
      和名字一样，这个语法分析器总是失败
      failure :: Parser a
failure = \inp -> []

    3. return
      这个语法分析器的功能是不消耗任何输出，始终返回v
      这个的定义不太一样了，它返回一个接受任何输入的一个语法分析器，这个语法分析器对于任意输入返回v。

      return :: a -> Parser a
return v = \inp -> [(v, inp)]

      这段代码根据上面对Parser的定义是能够运行的，因为

         Parser a
:: String -> [(a, String)]
:: [Char] -> [(Char, String)]

      但是在Parsing.hs下不能运行

      注意到在大多数语言中return是个命令，而Haskell中也不需要什么return，语句自身就是返回值，所以不要把两个混淆
      在Haskell的Prelude中也定义了return，它是类似于附带代码的实现

      > :t return
return :: Monad m => a -> m a

      对这样的定义先暂且不管

    4. +++
      这个语法分析器类似于一个选择结构，实际上它是一个存在的运算符<*>的重新实现。它表现得像语法分析器p，如果p能succeed，否则表现得像语法分析器q

      (+++) :: Parser a -> Parser a -> Parser a
p +++ q = \inp -> case p inp of
                        [] -> parser q inp
                        [(v, out)] -> [(v, out)]

    5. parse
      parse是个很重要的语法分析器，如果实际运行附带代码里面的Parsing.hs，就会发现

      print (item "123")

      是会报错的，正确的方式是运行

      print (parse item "123")

      这类似return的问题，这是因为Parser的定义不同导致的，在现阶段应该使用视频中给出的代码
      再看一下parse的定义

      parse :: Parser a -> String -> [(a, String)]
parse p inp = p inp

      值得注意的是parse函数将这个语法分析器（再次强调根据type Parser定义，Parse实际上也是一个函数）p应用(apply)到inp上。它的作用是让代码更清晰

  4. 检验上面的语法分析器
    检验上面的语法分析器需要Parser类型的定义。

    1. item
      > parse item ""
[]
> parse item "abc"
[('a', "bc")]
    2. failure
      > parse failure "abc"
[]
    3. return
      > parse (return 1) "abc"
[(1, "abc")]
    4. +++
      > parse (item +++ return 'd') "abc"
[('a', "bc")]
> parse (failure +++ return 'd') "abc"
[('d', "abc")]

  5. Sequencing
    到目前为止，我们都是使用递归来描述一系列的操作的，借助list comprehension或者高阶函数也可以实现循环操作，借助if、guard或者case实现选择语句。
    但是顺序操作，这个命令式语言最常见的操作却一直没有办法实现，事实上由于haskell的lazy特性，其更倾向于是并举的。而Monad正是解决这个问题的

    1. do
      可以用do来表示顺序操作，do能够作用在任意的monadic类型上

      p :: Parser (Char, Char)
p = do x <- item
        item
        y <- item
        return (x, y)

      但实际上这段代码只能够通过Parsing.hs运行！
      如果仍然定义

      type Parser a = String -> [(a, String)]

      简单地推导下可以发现

      :: Parser (Char, Char)
:: [((Char, Char), String)]

      但是返回的(x, y)是

      :: [(([(Char, String)], [(Char, String)]), String)]

      其中定义和<-这个运算符有关

    2. >>=
      这个操作称为bind，在C#中称为Select Many，先查看它的定义

      (>>=) :: Monad m => m a -> (a -> m b) -> m b

      因为Parser是一个Monad，为了直观一点，所以对应到Parser类型

      (>>=) :: Parser a -> (a -> Parser b) -> Parser b

      这个运算接受一个参数Parser a，将它给一个函数(a -> Parser b)，随后得到结果Parser b
      这其实就是讲操作序列化。是Monad的基本操作

    3. Monad
      根据Wikipedia，Monad具有两个操作，第一个是之前看到的return，另一个是上面的bind

      1. return
        return操作又称为unit，从一个普通类型a构造一个具有类型M a的monadic value
      2. bind
        这个操作接受一个M a的monadic value并且通过函数(a -> M b)来转换这个值到具有类型M b的另一个值

      此外，之前看到的failure其实都可以作为Monad的一部分，除此之外还有第四个运算符>>。这个运算符实际上是>>=的一个简化版本

      (>>) :: m a -> m b -> m b

      这个操作的意义实际上也是合成两个操作

    4. <-
      在list comprehension中出现了<-这个运算符

      [x | x <- xs]

      在这里，<-从[a]中 “提取” 了a
      但是对于do中的x <- item，item是上面刚提到的那个语法分析器，是一个Parser Char，通过<-得到Char类型，这是为什么？
      原来在do语句中，<-运算符另有作用，它称为single assignment operator，或binding operator。这个被用作在monadic computation中提取一个值，更准确的说，<-将Monad内的一个计算结果和一个名字绑定。例如这里的item是一个Parser a，在提取后得到一个a。

    5. 结论
      可以看出，这四行首先调用item获得一个x，然后再次调用item，但不关心结果，然后再次调用item，获得一个y。最后返回一个tuple(x, y)
      这有的类似于Linq中的from-select notation

      from x in item
from dummy in item
from y in item
select new {x, y}

  7. Monad的数学性质

    1. 对于Monad有三个规律

      1. do {e} -> e
        一个操作用do包裹起来等于它自己
      2. do {e; es} -> e >> do{es}
      3. do {let decls; es} -> let decls in do {es}

    2. Monad必须满足

      1. return a >>= f = f a
      2. f >>= return = f
      3. f >>= (\x -> g x >>= h) ≡ (f >>= g) >>= h
